# Projet : Site Web Statique HTML/CSS

Ce projet a pour but de mettre en pratique les compétences acquises lors du cours d'introduction à HTML et
CSS vu à l'IETC dans le cadre de mon bachelier en informatique de gestion.
Il s'agit d'un **site web canin humoristique**. Une de press people pour les chiens

---

## Cahier des charges

1. **Structure du site**
   - Minimum 5 pages HTML :
     - `index.html` : page d'accueil
     - `page2.html`, `page3.html`, `page4.html`, `page5.html` : pages thématiques
   - Thème : site web canin humoristique 'people'.

2. **Navigation**
   - Barre de navigation présente sur toutes les pages.
   - Navigation claire et intuitive.
   - Page active mise en évidence visuellement.

3. **Sémantique HTML**
   - Balises utilisées : `<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<aside>`, `<footer>`.
   - Balises de titre `<h1>` à `<h6>` utilisées hiérarchiquement.
   - Attributs `alt` pour toutes les images pour l’accessibilité.

4. **Mise en page**
   - Utilisation de **Flexbox** et **CSS Grid** pour organiser le contenu.
   - Responsive design : site adaptable à mobile (<768px), tablette (768–1024px) et desktop (>1024px).

5. **Formulaire**
   - Formulaire présent sur au moins une page.
   - Au moins 3 types de champs différents parmi : texte, email, téléphone, date, textarea, select, radio, checkbox.
   - Labels appropriés et champs `required` ou `placeholder` quand pertinent.
   - Bouton de soumission stylisé avec CSS.

6. **Organisation CSS**
   - Fichiers CSS externes (`style.css`, optionnel `responsive.css`).
   - Styles regroupés par composant ou page.
   - Noms de classes significatifs et commentaires si nécessaire.

## Fonctionnalités principales

- Header et navigation fixe et intuitive
- Footer avec contact et liens vers réseaux sociaux
- Contenu humoristique sur toutes les pages
- Formulaire fonctionnel avec différents types de champs
- Mise en page responsive et esthétique
- Utilisation de Flexbox et CSS Grid pour organiser les sections

## Lancer le site

1. Télécharger ou cloner le projet.  
2. Ouvrir `index.html` dans un navigateur web.  
3. Naviguer entre les pages pour découvrir le contenu humoristique.
